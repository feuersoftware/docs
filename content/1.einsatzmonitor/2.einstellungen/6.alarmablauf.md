# Alarmablauf

![EinsatzMonitor Alarmablauf](/images/einsatzmonitor/EinsatzMonitor_Alarmablauf.png)

Der Alarmablauf ermöglicht es, auf die Verarbeitung eines Einsatzes Einfluss zu nehmen und bestimmte Aktionen in den Alarmablauf zu integrieren. Aktionen werden auf der rechten Seite ausgewählt und auf der linken Seite in den Alarmablauf eingefügt – mit festgelegter zeitlicher Abfolge.

## Verfügbare Plugins

### Text löschen

Sucht nach einem Text und löscht ihn aus der Einsatzmeldung.

| Parameter | Beschreibung |
|---|---|
| **Start** | Text, ab dem gelöscht wird (inklusive) |
| **Stopp** *(optional)* | Text, bis zu dem gelöscht wird (exklusive). Fehlt Stopp, wird bis zum Ende gelöscht. |

### Text ersetzen

Sucht nach einem Text in einer Variable und ersetzt ihn durch einen definierten Text.

**Beispiel:** „Str" automatisch durch „Straße" ersetzen.

### Einsatzverwaltung

Bricht den Alarmablauf ab oder schließt einen Einsatz, wenn ein bestimmter Text gefunden wird. Wirkt nur auf bestehende Einsätze (Referenz: Einsatznummer). Als Fallback wird die „Anzeige säubern"-Zeit (Standard: 15 Minuten) verwendet.

| Parameter | Beschreibung |
|---|---|
| **Variable** | Wo gesucht wird: gesamte Einsatzmeldung, Standardvariable oder eigenes Zusatzfeld |
| **Filter** | Text, nach dem gesucht wird (Enthält-Prüfung) |
| **Connect: Einsatz** | **Schließen** – Einsatz erfolgreich abgeschlossen; **Abbrechen** – Einsatz schließen + Push-Nachricht „Einsatz Abbruch" an alarmierte Kräfte |

### Koordinaten

Fügt einem Einsatz Koordinaten hinzu, wenn ein bestimmter Text in Adresse oder Objekt gefunden wird.

| Parameter | Beschreibung |
|---|---|
| **Variable** | Adresse oder Objekt |
| **Filter #1** | Pflichttext in der Variable |
| **Filter #2** *(optional)* | Zweiter Text (Reihenfolge egal: „Filter1 Filter2" oder „Filter2 Filter1") |
| **Position (Lat)** | Breitengrad |
| **Position (Lng)** | Längengrad |

### Whitelist

Bricht den Alarmablauf ab, wenn ein bestimmter Text **nicht** gefunden wird. Nützlich, um Zusatzalarmierungen nur bei bestimmten Einsatzarten auszulösen.

| Parameter | Beschreibung |
|---|---|
| **Variable** | Wo gesucht wird: gesamte Einsatzmeldung, Standardvariable oder Zusatzfeld |
| **Whitelist Text** | Text, der enthalten sein muss |
| **Vor Connect?** | **Ja** – Plugin läuft vor Übermittlung an Connect; **Nein** – Plugin läuft danach |

### Drucken

Druckt bei Alarmeingang automatisch Einsatzaufträge auf einem konfigurierten Drucker. Weitere Informationen im HowTo [Drucken](../3.howtos/3.drucken).

| Parameter | Beschreibung |
|---|---|
| **Druckername** | Drucker auswählen |
| **Anzahl** | Anzahl der Ausdrucke oder **AAO** (Anzahl richtet sich nach alarmierten Fahrzeugen) |
| **Template** | Dokumentvorlage für das Drucklayout |

### Programm starten

Führt ein Programm oder Skript auf dem EinsatzMonitor-Rechner aus. Dem Programm können Einsatzvariablen als Parameter übergeben werden.

| Parameter | Beschreibung |
|---|---|
| **Programmpfad** | Speicherort der ausführbaren Datei |
| **Parameter** | Parameter an das Programm (Variablen s. u.) |

**Verfügbare Variablen:**

```
%EINSATZSTICHWORT%
%EINSATZSTICHWORTUEBERSETZUNG%
%EINSATZNUMMER%
%EINSATZBEGINN%
%SACHVERHALT%
%ADRESSE%
%OBJEKT%
%RIC%
%GMA%
%SEGMENT%
%ZUSATZFELD1%
%ZUSATZFELD2%
%ZUSATZFELD3%
%GOOGLEMAPS%
```

**PowerShell-Skripte ausführen:**

Standardmäßig sind `.bat`-Dateien vorgesehen. PowerShell-Skripte können mit einem Execution-Policy-Bypass aufgerufen werden:

| Feld | Wert |
|---|---|
| **Programmpfad** | `powershell.exe` |
| **Parameter** | `-ExecutionPolicy Bypass -File C:\temp\script.ps1` |

### Einsatzauftrag vorlesen

Gibt Einsatzinformationen bei Alarmeingang akustisch über angeschlossene Lautsprecher aus (Text-to-Speech).

| Parameter | Beschreibung |
|---|---|
| **Stimme** | Unter Windows installierte TTS-Stimmen |
| **Startverzögerung** | Wartezeit in Sekunden vor der Ausgabe |
| **Wiederholungen** | Anzahl der Wiederholungen |
| **Pause** | Pausenzeit in Sekunden zwischen Wiederholungen |
| **Geschwindigkeit** | Lesegeschwindigkeit (langsamer / schneller) |
| **Alarmtext** | Auszugebender Text mit Variablen |

### Zusatzalarmierung auslösen

Löst eine konfigurierte Zusatzalarmierung im Alarmablauf aus. Wird benötigt, wenn eine Zusatzalarmierung über den EinsatzMonitor eingerichtet ist.

### Wake On LAN

Schaltet netzwerkfähige Geräte im gleichen Netzwerk per Wake-on-LAN ein. Beispiel: PC auf ELW oder Einsatzleitstand automatisch hochfahren.

### Sound abspielen

Spielt eine Audiodatei ab – typischerweise ein Alarmgong.

::callout{type="info"}
Es werden ausschließlich **`.wav`-Dateien** unterstützt. Andere Formate können mit einem Konvertierungstool (z. B. MP3 zu WAV) umgewandelt werden.
::

### Add Zusatzinformationen mit CSV

Sucht in einer eigenen CSV-Datei nach einem Schlüsselwert (Spalte 1, z. B. eine GMA-Nummer) und fügt den zugehörigen Wert aus Spalte 2 automatisch als Zusatzinformation zum Einsatz hinzu.

### Neuer Route Startpunkt

Ändert den Startpunkt der Routenberechnung, z. B. auf den Eingang einer Ortschaft.

| Parameter | Beschreibung |
|---|---|
| **Feld** | Ort oder Ortsteil |
| **Gleich** | Name des Ortes, für den der Startpunkt gesetzt wird |
| **Position (Lat)** | Breitengrad des Startpunkts |
| **Position (Long)** | Längengrad des Startpunkts |

### Verzögerung

Verzögert den weiteren Alarmablauf um eine konfigurierte Zeit.

### UI Reload

Lädt die Benutzeroberfläche neu, um z. B. Darstellungsprobleme bei der Karte zu beheben.

### Unterbrechung

Unterbricht den Alarmablauf unter einer definierten Bedingung. Der Monitor wechselt dabei nicht in den Standby-Modus – nachfolgende Plugins werden lediglich blockiert.

| Parameter | Beschreibung |
|---|---|
| **Variable** | Variable, auf die die Bedingung angewandt wird |
| **Text in Variable** | Bedingungstext |
