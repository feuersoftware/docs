# TetraControl Verbindung

Über diese Einstellungen kann der EinsatzMonitor direkt mit einer [TetraControl](https://tetracontrol.de/)-Instanz verbunden werden. TetraControl wertet Daten von angeschlossenen TETRA-Funkgeräten oder Pagern aus und stellt diese über einen Webserver bereit. Der EinsatzMonitor empfängt Status- und Positionsupdates der Fahrzeuge sowie Benutzerrückmeldungen und kann diese an Connect weiterleiten.

> **Hinweis:** Für Connect-Kunden wird die Nutzung von [TetraControl2Connect](/tools/tetracontrol2connect) empfohlen, da dieses Tool mehrere Standorte unterstützt und eine Admin-Oberfläche bietet. Die direkte EinsatzMonitor-Anbindung eignet sich besonders für Installationen ohne Connect.

Nach erfolgreicher Konfiguration erscheint in der Symbolleiste des EinsatzMonitors ein **grünes Funkgerät-Symbol**, das die aktive Verbindung zu TetraControl signalisiert.

---

## Verbindungseinstellungen

| Einstellung | Beschreibung |
|---|---|
| **Aktiv** | TetraControl-Verbindung an oder aus. Der EinsatzMonitor muss nach einer Änderung neu gestartet werden. |
| **Webserver Name / IP** | FQDN oder IP-Adresse des TetraControl Servers (kann auch lokal laufen) |
| **Webserver Port** | Port des TetraControl Webservers |
| **Webserver Benutzer** | Benutzername (wird in TetraControl angelegt) |
| **Webserver Passwort** | Passwort des Webserver-Benutzers |

---

## Verarbeitungseinstellungen

| Einstellung | Beschreibung |
|---|---|
| **Alarmgeber ISSI** | ISSI, auf die reagiert werden soll. Leer lassen, um auf alle ISSIs zu reagieren. |
| **Fahrzeug Update** | Fahrzeugstatus aus TetraControl berücksichtigen: Ja / Nein |
| **Force Reconnect** | Wenn länger als 60 Minuten keine Kommunikation stattgefunden hat, automatisch neu verbinden. Nützlich bei Anbindung über Internet oder VPN. |
| **Nutze CalloutReferenz** | Alarm-SDS werden mehrfach versendet. Nachfolgende SDS-Nachrichten mit gleicher CalloutReferenz werden ignoriert. |
| **SDS Zusammenfassen** | Alle SDS-Nachrichten innerhalb einer konfigurierten Zeitspanne zu einer zusammenfassen. |
| **Zusammenfassen Zeitspanne** | Zeitspanne (in Sekunden) für das Zusammenfassen von SDS-Nachrichten. |
| **Subric Filter** | Nur auf SDS reagieren, die an die angegebenen Subrics gesendet wurden. Mehrere Subrics durch ein beliebiges Trennzeichen trennen (z. B. Leerzeichen, Semikolon, Komma). Beispiel: `11;15;31;37` |
| **Subric Filter (Verknüpfe mit)** | Verschiedene Subrics verknüpfen – z. B. eine Subric zur Alarmierung und eine für Navidaten. Der EinsatzMonitor fasst diese zusammen. |
| **Einsatzauftrag senden bei \* Taste** | Reagiert auf die \*-Taste der hinterlegten Fahrzeuge und antwortet per SDS mit dem aktuellen Einsatzauftrag. |
| **Einsatzauftrag Text** | Text, der per SDS als Einsatzauftrag gesendet wird. |
| **Connect Status 5 unterdrücken** | Fahrzeugstatus 5 wird nicht an Connect gesendet, auch wenn die Fahrzeugstatus-Auswertung aktiv ist. |

## Programm bei Statusänderung ausführen

Diese Funktion ermöglicht die Ausführung eines externen Programms oder Skripts, wenn eine bestimmte Statusfolge von TetraControl empfangen wird – z. B. um ein Tor zu öffnen, wenn ein bestimmter Status gedrückt wird.

| Einstellung | Beschreibung |
|---|---|
| **Status** | Der Status oder die Statusfolge, auf die reagiert werden soll |
| **ISSI** | Die ISSI, von der der Status gesendet wird. Damit lässt sich z. B. festlegen, dass nur das HLF Tor 1 öffnen darf. |
| **Ausführen** | Pfad zum Programm oder Skript, das ausgeführt werden soll (`.bat`-Dateien werden unterstützt) |
| **SDS Rückmeldung** | Sendet bei Ausführung eine SDS-Nachricht als Erfolgsmeldung |
